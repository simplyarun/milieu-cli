<objective>
Fix the build pipeline so that compiled output excludes test files, and add automated packaging tests.

Purpose: The current `tsc` build compiles all `.ts` files including `__tests__/` directories into `dist/`, inflating the npm package from ~80 files to 275 files (578 KB). This plan creates a separate build tsconfig that excludes tests, updates the build script to clean and rebuild, adds a prepublishOnly lifecycle hook, and writes automated tests verifying these packaging invariants.

Output: `tsconfig.build.json`, updated `package.json` scripts, packaging test file.
</objective>

1. Create `tsconfig.build.json` at project root:
```json
{
  "extends": "./tsconfig.json",
  "exclude": [
    "src/**/__tests__/**",
    "src/**/*.test.ts",
    "src/**/*.spec.ts"
  ]
}
```
This extends the main config (keeping all compilerOptions) but excludes test directories and test file patterns from compilation.

2. Update `package.json` scripts section to:
```json
{
  "scripts": {
    "build": "rm -rf dist && tsc -p tsconfig.build.json",
    "dev": "tsx src/index.ts",
    "test": "vitest run --reporter=verbose",
    "typecheck": "tsc --noEmit",
    "prepublishOnly": "npm run build",
    "prepack": "npm run build"
  }
}
```
Changes:
- `build` now cleans dist first (`rm -rf dist`) and uses `tsconfig.build.json` instead of bare `tsc`
- Added `prepublishOnly` to auto-build before `npm publish`
- Added `prepack` to auto-build before `npm pack`
- `typecheck` remains `tsc --noEmit` (still uses main tsconfig so it typechecks tests too)

3. Run `npm run build` to produce clean dist output.

4. Run `npm pack --dry-run` and verify:
   - Zero lines containing `__tests__`
   - Total file count under 100
   - No `.test.js` or `.test.d.ts` files
