    **llms-txt.ts** -- exports `checkLlmsTxt(baseUrl: string, timeout?: number): Promise<Check>` and `checkLlmsFullTxt(baseUrl: string, timeout?: number): Promise<Check>`:

    checkLlmsTxt:
    - httpGet(`${baseUrl}/llms.txt`, { timeout })
    - If !result.ok: return `{ id: "llms_txt", label: "llms.txt", status: "fail", detail: "No llms.txt found" }`
    - If ok but body.trim().length === 0: same fail
    - Calculate sizeBytes: `new TextEncoder().encode(result.body).byteLength`
    - Extract firstLine: `result.body.trim().split("\n")[0].trim()`
    - If firstLine matches `/^#\s+\S/`: return pass with `detail: "llms.txt found (${sizeBytes} bytes)"`, `data: { sizeBytes, firstLine }`
    - Else: return partial with `detail: "llms.txt found but missing H1 header"`, `data: { sizeBytes, firstLine }`

    checkLlmsFullTxt:
    - httpGet(`${baseUrl}/llms-full.txt`, { timeout })
    - If !result.ok or empty body: return `{ id: "llms_full_txt", label: "llms-full.txt", status: "fail", detail: "No llms-full.txt found" }`
    - Else: return pass with `detail: "llms-full.txt found (${sizeBytes} bytes)"`, `data: { sizeBytes }`
    - No partial state. No H1 requirement.

    For pass results, set `data: { detail: string }` matching the detail text fragment.

    **json-ld.ts** -- exports `checkJsonLd(html: string): Check` and a type `JsonLdBlock`:

    ```typescript
    export interface JsonLdBlock {
      type: string | string[];
      context: unknown;
    }

    export function checkJsonLd(html: string): Check;
    ```

    Implementation:
    - Regex: `/<script\s+type=["']application\/ld\+json["'][^>]*>([\s\S]*?)<\/script>/gi`
    - Execute regex on html, collect all matches
    - For each match, try JSON.parse(match[1])
    - Handle arrays: if parsed is Array, iterate items; else treat as single object
    - For each item: if typeof item === "object" && item !== null && "@type" in item, push to blocks array as `{ type: item["@type"], context: item["@context"] ?? null }`
    - If blocks.length === 0: return `{ id: "json_ld", label: "JSON-LD Structured Data", status: "fail", detail: "No JSON-LD structured data found" }`
    - If blocks.length === 1: build types string from first block's type (if array, join with ", ")
      - detail: `"1 JSON-LD block found: ${typesStr}"`
    - If blocks.length > 1: collect all type strings, join with ", "
      - detail: `"${blocks.length} JSON-LD blocks found: ${typesStr}"`
    - Return pass with `data: { blocks }`

    Helper function to flatten type to string: if string[], join with ", "; if string, return as-is.

    **schema-org.ts** -- exports `checkSchemaOrg(html: string, jsonLdCheck: Check): Check`:

    Takes the JSON-LD Check result (from checkJsonLd) to avoid re-parsing. Also scans HTML for Microdata.

    Implementation:
    - Initialize `types: string[]` and `sources: ("json-ld" | "microdata")[]`
    - Check JSON-LD: if jsonLdCheck.status === "pass" and jsonLdCheck.data has blocks array, iterate blocks. For each block, check if block.context is a string containing "schema.org" OR is an array containing a string with "schema.org". If yes, extract type name(s) and add to types. If any found, push "json-ld" to sources.
    - Check Microdata: regex `itemtype=["'](https?:\/\/schema\.org\/[^"']+)["']` with `gi` flag. For each match, extract type name (last path segment via `.split("/").pop()`). Deduplicate. If any found, push "microdata" to sources.
    - If types.length === 0: return `{ id: "schema_org", label: "Schema.org Markup", status: "fail", detail: "No Schema.org markup found" }`
    - Deduplicate types array.
    - Build detail string:
      - sources has both "json-ld" and "microdata": `"Schema.org types found via JSON-LD and Microdata: ${types.join(", ")}"`
      - sources has only "json-ld": `"Schema.org types found via JSON-LD: ${types.join(", ")}"`
      - sources has only "microdata": `"Schema.org types found via Microdata: ${types.join(", ")}"`
    - Return pass with `data: { types, sources }`

    For tests: these are pure functions, no mocking needed. Pass HTML strings directly.
