**Exported types:**
```typescript
export interface RobotsTxtResult {
  parseable: boolean;
  ruleCount: number;
  groups: RobotsGroup[];
  sitemaps: string[];
}
export interface RobotsGroup {
  userAgents: string[];     // Lowercased for matching
  rules: RobotsRule[];      // In document order
}
export interface RobotsRule {
  type: "allow" | "disallow";
  path: string;             // Raw path pattern
}
```

**Exported functions:**
- `parseRobotsTxt(content: string): RobotsTxtResult` -- parses raw robots.txt text
- `matchesPath(pattern: string, path: string): boolean` -- converts robots pattern to regex

**parseRobotsTxt algorithm:**
1. Strip UTF-8 BOM (`\xEF\xBB\xBF`) from start if present
2. Split on `\r\n`, `\r`, or `\n` (use `.split(/\r\n|\r|\n/)`)
3. For each line: strip comment (everything after `#`), trim whitespace, skip empty
4. Parse directive: split on first `:`, lowercase the directive name
5. Track state: `currentGroup: { userAgents: string[], rules: RobotsRule[] } | null`
6. On `user-agent`: if currentGroup has rules (meaning we already saw Allow/Disallow), push currentGroup to groups and start a new one. Otherwise, add to currentGroup.userAgents (consecutive User-agent lines = same group). Lowercase the value.
7. On `allow` / `disallow`: if no currentGroup, create one with userAgents []. Add rule with trimmed path value (may be empty string for empty Disallow).
8. On `sitemap`: push value to sitemaps array (not tied to any group).
9. On anything else: ignore.
10. After loop: push final currentGroup if it exists.
11. Set `ruleCount` = sum of `group.rules.length` for all groups.
12. `parseable` = true (if we got here without throwing).

**matchesPath algorithm:**
1. If pattern is empty string, return true (empty Disallow matches everything).
2. Escape all regex special chars EXCEPT `*` and trailing `$`.
3. Replace `*` with `.*`.
4. If pattern ends with `$`, anchor the regex at end; otherwise the pattern is a prefix match (no anchor).
5. Return `new RegExp("^" + regexStr).test(path)`.

For regex escaping, escape these characters: `\ . + ? [ ] ( ) { } | ^ /` -- but NOT `*` (handled separately) and NOT `$` at end (handled separately).

**Per-crawler evaluation algorithm:**
1. Find groups matching this crawler's userAgent (case-insensitive match on group.userAgents)
2. If no specific group, fall back to the group with userAgent `"*"`
3. If no matching group at all → "pass" (no restrictions)
4. Get rules from the matching group (prefer specific group over * group)
5. Evaluate policy from the rules:
