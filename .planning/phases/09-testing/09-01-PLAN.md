**parseRobotsTxt -- add these new tests inside the existing describe block:**

1. "handles whitespace around colon in directive" -- input `User-agent : *\nDisallow : /secret` should parse group with userAgent `*` and rule path `/secret`
2. "handles blank lines between groups" -- two groups separated by blank lines: `User-agent: a\nDisallow: /x\n\n\nUser-agent: b\nDisallow: /y` should produce 2 groups
3. "handles comment-only lines between directives" -- `User-agent: *\n# Block the secret\nDisallow: /secret` should parse 1 group with 1 rule
4. "handles very long robots.txt with 50+ rules" -- generate content with User-agent: * and 50 Disallow rules /path-0 through /path-49. Verify ruleCount is 50
5. "handles rules without preceding User-agent" -- `Disallow: /orphan` should still create a group (with empty userAgents array) containing the rule
6. "handles consecutive Sitemap lines mixed with groups" -- `Sitemap: https://a.com/s.xml\nUser-agent: *\nDisallow: /\nSitemap: https://b.com/s.xml` should produce 2 sitemaps and 1 group
7. "handles mixed case Sitemap directive" -- `SITEMAP: https://example.com/sitemap.xml` should extract sitemap URL (case-insensitive directive parsing)
8. "handles tab characters as whitespace in values" -- `User-agent:\t*\nDisallow:\t/path` should parse correctly

**matchesPath -- add these new tests inside the existing describe block:**

9. "matches wildcard in middle of path" -- `matchesPath("/foo/*/bar", "/foo/anything/bar")` should be true
10. "matches multiple wildcards" -- `matchesPath("/a/*/b/*", "/a/x/b/y")` should be true
11. "matches file extension wildcard" -- `matchesPath("/*.json", "/data.json")` should be true
12. "matches file extension wildcard with $ anchor" -- `matchesPath("/*.json$", "/data.json")` should be true, but `matchesPath("/*.json$", "/data.json/extra")` should be false
13. "matches path with query string" -- `matchesPath("/search", "/search?q=test")` should be true (prefix match)
14. "does not match when pattern has more segments" -- `matchesPath("/a/b/c", "/a/b")` should be false
