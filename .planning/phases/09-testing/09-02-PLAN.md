**Step 1: Create fixture file at `src/bridges/__tests__/fixtures/example-com.ts`**

This file exports recorded HTTP responses for a fictional "example.com" scan. All data is hardcoded -- no network calls. The fixture maps URL patterns to HttpSuccess/HttpFailure responses.

Create a `createFixtureResponder` function that takes a map of URL-pattern to HttpResponse and returns a function `(url: string, options?: any) => Promise<HttpResponse>` that looks up the URL in the map. If no match, return a 404 HttpFailure.

Define two fixture sets:

**Fixture A: "healthy-site"** -- a site that passes most checks:
- `https://example.com` (GET) -> 200, body: realistic HTML with `<head>` containing a JSON-LD block `{"@context":"https://schema.org","@type":"Organization","name":"Example Corp"}` and a meta robots `<meta name="robots" content="index, follow">`, headers: `content-type: text/html; charset=utf-8`
- `https://example.com` (HEAD) -> 200, empty body, headers: `content-type: text/html`
- `https://example.com/robots.txt` -> 200, body: `User-agent: *\nAllow: /\n\nUser-agent: GPTBot\nDisallow: /private/\n\nSitemap: https://example.com/sitemap.xml`, headers: `content-type: text/plain`
- `https://example.com/openapi.json` -> 200, body: `{"openapi":"3.1.0","info":{"title":"Example API"},"paths":{"/users":{},"/items":{}}}`, headers: `content-type: application/json`
- `https://example.com/.well-known/security.txt` -> 200, body: `Contact: security@example.com\nExpires: 2027-01-01T00:00:00.000Z`, headers: `content-type: text/plain`
- `https://example.com/llms.txt` -> 200, body: `# Example Corp\n> AI-friendly documentation index`, headers: `content-type: text/plain`
- All other URLs -> 404 HttpFailure

**Fixture B: "minimal-site"** -- a site that mostly fails:
