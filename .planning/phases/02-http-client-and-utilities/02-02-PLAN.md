**Step 2: Create `src/utils/http-client.ts`** with these exports:

1. `HttpGetOptions` interface:
```typescript
export interface HttpGetOptions {
  method?: "GET" | "HEAD";
  timeout?: number;        // default 10000
  maxRedirects?: number;   // default 5
  maxBodyBytes?: number;   // default 5 * 1024 * 1024 (5MB)
  dnsCache?: DnsCache;     // scan-scoped, caller creates Map
  headers?: Record<string, string>;
}
```

2. `httpGet(url: string, options?: Partial<HttpGetOptions>): Promise<HttpResponse>` -- the main export. Implementation:

   a. Merge options with defaults: `{ method: "GET", timeout: 10_000, maxRedirects: 5, maxBodyBytes: 5 * 1024 * 1024, dnsCache: new Map() }`

   b. Record start time with `performance.now()`

   c. Call internal `fetchWithRetry(url, mergedOptions)` which handles the retry wrapper

   d. Return HttpResponse with `durationMs: Math.round(performance.now() - start)`

3. Internal `fetchWithRetry` function:
   - Call `fetchOnce(url, options)`
   - If result is retriable (timeout, connection_refused, or ok:true with status >= 500) AND retry hasn't been attempted: wait 2000ms (`await new Promise(r => setTimeout(r, 2000))`), call fetchOnce again
   - Do NOT retry on: dns, ssrf_blocked, ssl_error, http_error (4xx), bot_protected, body_too_large, unknown

4. Internal `fetchOnce` function -- the core logic:
   a. Parse URL with `new URL(url)` -- catch TypeError, return `{ ok: false, error: { kind: "unknown", message: "Invalid URL", url } }`
   b. SSRF pre-flight: call `validateDns(hostname, dnsCache)`. If `safe: false`, return `{ ok: false, error: { kind: "ssrf_blocked", message: result.error, url } }`. If DNS fails (error message starts with "DNS resolution failed"), return kind `"dns"` instead.
   c. Set up headers: `{ "User-Agent": "milieu-cli/0.1.0", ...options.headers }`
   d. Call `fetch(currentUrl, { method, redirect: "manual", signal: AbortSignal.timeout(remainingTimeout), headers })`
      - Add code comment at `redirect: "manual"` line: `// Manual redirects: required for SSRF re-validation at each hop and redirect chain tracking`
   e. Redirect loop (for status 300-399):
      - Get `location` header from response
      - If no location header: break out, return current response as success
      - Call `resolveRedirectUrl(location, currentUrl)` -- if `ok: false`, return HttpFailure with kind `"http_error"`, message about invalid redirect
      - SSRF re-validate the new hostname via `validateDns`
      - Add currentUrl to redirects array
      - Increment redirect counter, if exceeds maxRedirects return HttpFailure kind `"http_error"` message `"Too many redirects (max ${maxRedirects})"`
      - Update currentUrl, continue loop
   f. Bot protection detection on final response:
      - 403 + server header containing "cloudflare" (case-insensitive) -> bot_protected
      - 403 + `cf-ray` header present -> bot_protected
      - 429 (any server) -> bot_protected
      - 503 + server header containing "cloudflare" -> bot_protected
      - If detected: return `{ ok: false, error: { kind: "bot_protected", message: "Bot protection detected", statusCode: status, url } }`
   g. For 4xx/5xx (non-bot): return HttpFailure with kind `"http_error"`, statusCode, message including status text
   h. Body reading for 2xx success:
      - Check Content-Length header: if present and > maxBodyBytes, return HttpFailure kind `"body_too_large"`
      - Read body with `response.text()` -- if response body exceeds maxBodyBytes during read (check `body.length` after), truncate to maxBodyBytes
      - For HEAD method: body is always `""`
   i. Convert response.headers to `Record<string, string>` by iterating `response.headers.entries()` and building a plain object
   j. Return HttpSuccess: `{ ok: true, url: currentUrl, status, headers: headerRecord, body, redirects, durationMs: 0 }` (durationMs set by outer wrapper)

5. Internal `classifyFetchError(error: unknown, url: string): HttpFailure` function:
   - `error instanceof DOMException && error.name === "TimeoutError"` -> kind `"timeout"`
   - `error instanceof TypeError && error.cause`:
     - `cause.code === "ENOTFOUND"` -> kind `"dns"`
     - `cause.code === "ECONNREFUSED"` -> kind `"connection_refused"`
     - `cause.code` matching `"CERT_HAS_EXPIRED" | "DEPTH_ZERO_SELF_SIGNED_CERT" | "UNABLE_TO_VERIFY_LEAF_SIGNATURE" | "ERR_TLS_CERT_ALTNAME_INVALID"` -> kind `"ssl_error"`
   - Default: kind `"unknown"`, message `String(error)`
